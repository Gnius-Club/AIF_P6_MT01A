🎯 Objetivo de Aprendizaje

Los estudiantes podrán: Comprender la API como una caja de control y la complejidad de los sistemas robóticos. Usar Micro:Bit junto con XGO para realizar la conexión inicial y ejecutar comandos base de movimiento de forma segura.

🤖 Prompt de Referencia para XOLO (Solo para Maestro)

⚠️ NOTA: XOLO ya conoce el tema de esta misión. Este prompt es SOLO de referencia en caso de emergencia técnica: 
Quiero aprender cómo encender, conectar y ejecutar el primer movimiento seguro del XGO usando Python. Explícame qué es una API y por qué un comando como xgo.move_forward(speed) activa un sistema complejo de movimiento (15 DoF trabajando a la vez). Luego guíame paso a paso para que el robot se ponga de pie (xgo.stand()), se siente (xgo.sit()) y avance con velocidad controlada. Incluye un checklist de seguridad antes de ejecutar el código, y un checklist rápido de debugging si algo falla (conexión, indentación, sintaxis). Al final, evalúame con 3 preguntas rápidas para confirmar si entendí.

⚡ Puntos Esenciales de la Misión

🎓

Concepto Clave

API como caja de control: Es como un control remoto que le dice al robot qué hacer sin que nosotros tengamos que saber cómo mover cada uno de sus 15 motores por separado.

🔧

Habilidad Técnica

Usar comandos básicos para que el perro robot XGO se mueva y realice acciones simples.

🧪

Experimentación

Probar con diferentes velocidades y secuencias de movimiento para entender limitaciones.

💬

Interacción con XOLO

XOLO ya conoce a cada estudiante y el tema de hoy. Solo abren el chat y comienzan a aprender. Tú guías el timing y motivación.

🚨 Resolución Rápida de Problemas

  • Si los alumnos no pueden ingresar a Gnius Space: Activa XOLO SOS en xolo.gnius.club para continuar la misión sin interrumpir la clase.
  • Si el XGO no responde: Verificar que la batería esté cargada y el interruptor en ON. Revisar que el micro:bit esté bien insertado en la ranura.
  • Si el robot se mueve de forma extraña: Confirmar que la extensión XGO esté instalada y que los bloques de comando sean los correctos.
  • Si hay errores en el código: Recordar a los estudiantes que XOLO puede ayudar a depurar. Pídeles que copien el error y se lo muestren a XOLO.

📊 Diapositiva 3: Brain Spark

1. 💬 Pide a XOLO que te ayude a crear:

"Quiero idear un mini-proyecto donde el XGO sea mi “robot explorador” en una misión de bienvenida. Ayúdame a diseñar una secuencia de 4 acciones que se vean “inteligentes” (levantarse, mirar alrededor, avanzar, detenerse). Después genera el código completo en Python y sugiere cómo presentarlo como demostración al grupo, explicando que esto es el primer paso para dominar sistemas robóticos complejos antes de usar IA."

2. 🔬 Experimenta y descubre con tu programa:

  • ¿El robot realiza los movimientos en el orden correcto?
  • ¿Qué pasa si el robot intenta caminar sobre una superficie muy lisa o resbalosa?
  • ¿Funciona igual el comando de avance con velocidad 50 vs velocidad 100?
  • Si la API es como un control remoto, ¿qué pasaría si enviamos un comando que el robot no entiende?
  • Con 15 motores moviéndose a la vez, ¿por qué es importante que el sistema sea complejo antes de añadirle IA?

¡Es hora de crear! Vamos a convertir a nuestro XGO en un verdadero explorador. Primero, usen el prompt que ven en pantalla para que XOLO los ayude a idear una secuencia inteligente: levantarse, mirar, avanzar y detenerse. Una vez que tengan su GCS, tradúzcanlo a bloques en MakeCode. Prueben primero en el simulador y, cuando estén listos, carguen el código al micro:bit. Por ejemplo, podrían imaginar que el XGO está explorando un planeta nuevo y debe moverse con cuidado. No olviden responder las preguntas de experimentación para ver cómo se comporta su robot en el mundo real. Si tienen errores o dudas, pregúntenle a XOLO, él los ayudará a depurar.
